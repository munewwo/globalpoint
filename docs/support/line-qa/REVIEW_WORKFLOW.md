# 校正（レビュー）と誤回答の修正フロー

目的は「誤回答を隠さずに修正し、同じ誤りを繰り返さない」ことです。LINEのやり取りは短文になりがちなので、**根拠**と**注意点**をアーカイブ側に残します。

## 1. アーカイブの記録

- すべてのQ&Aは `archive/` に「1件=1ファイル」で残します。
- 返信後すぐに、最低限次を記録します。
  - 質問原文
  - 送信した回答
  - 参照した文書（パス）
  - 追加で確認が必要な点（あれば）

テンプレ: `templates/archive-entry-template.md`

## 2. レビュー（校正）

### レビュー観点（チェックリスト）

- **結論が先にあるか**
- **重要点は3点以内か**
- **手順が必要なら手順で案内できているか**
- **法的断定の表現がないか**
- **参照先（根拠）が適切か（1〜2件に絞れているか）**
- **用語の表記ゆれがないか**（必要なら `docs/terms/README.md` を更新）
- **エスカレーションすべき内容を通常回答していないか**

### ステータス運用

アーカイブの `status` を次で運用します。

- `draft`: 記録したが未レビュー
- `reviewed`: レビュー済み（問題なし／軽微修正のみ）
- `corrected`: 誤りがあり修正した
- `archived`: 参照用として固定（後からの追記は「修正履歴」に残す）

## 3. 誤回答が見つかった場合（指摘→修正）

1. **影響範囲の判断**
   - 誤情報により、加盟クラブの運用や顧客説明に影響が出る可能性があるか
   - 法令・報道・個人情報・規約解釈・詐欺に関わるか（関わる場合は即エスカレーション）
2. **正しい回答案の作成**
   - 根拠文書（パス）を明記
   - 法的断定は避け、「当社の運用（判断）として」を用いる
3. **アーカイブの更新**
   - `status` を `corrected` に変更
   - 「校正・修正履歴」に理由と修正内容を残す
4. **必要なら加盟クラブへフォロー**
   - 重大な誤りであれば、短く訂正し、次に必要な対応を案内する

## 4. 再発防止（仕組み）

- **用語・表記**: 表記ゆれや未定義語は `docs/terms/README.md` に集約
- **参照先の固定**: よく参照する文書は `docs/support/line-qa/README.md` と `README.md` にリンクを置く
- **テンプレ改善**: よく起きる誤りがあれば、テンプレやチェックリストに追記して運用で潰す
