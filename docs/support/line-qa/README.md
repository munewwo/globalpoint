# LINE QA運用ガイド（加盟クラブ向け）

このガイドは、加盟クラブ（カスタマー）からの問い合わせに対し、**簡潔**かつ**重要点中心**で回答し、必要に応じて次のサポートを提案するための運用ルールです。

## 目的

- LINE公式アカウントでの回答品質を揃える
- 法的断定を避けつつ、当社の運用・判断として分かりやすく伝える
- Q&Aをアーカイブし、誤回答があれば指摘・修正して再発防止につなげる

## 基本方針（必ず守る）

- **結論を先に**: まず1〜2行で結論を伝える（相手が最初の数行だけ読んでも理解できる）。
- **重要点は最大3点**: 重要なポイントが少ない場合は、結論の次に箇条書きで要点だけを伝える。
- **手順があるなら手順で案内**: 順番に進める必要がある対応は、番号付きで案内する。
- **提案は必要なときだけ**: 追加案内が必要そう／やった方が良い次アクションがある場合のみ、最後に提案する。
- **不確実なことは断定しない**: 「法的に問題ない」「必ず大丈夫」等の断定は避け、当社の取り組み・判断として表現する。

## 文体・言い回し

### トーン

- **丁寧だが柔らかい**（圧をかけない、相手を否定しない、短く区切る）

### 推奨フレーズ（例）

- 「結論からお伝えしますと、〜です。」
- 「当社の現時点の運用（判断）としては、〜の整理です。」
- 「念のため、次の点をご確認ください。」
- 「もし差し支えなければ、状況確認のために〜を教えてください。」
- 「（必要に応じて）こちらで確認・ご案内しますので、〜をご共有いただけますか。」

### 避ける表現（NG）

- 「法的に問題ありません」「合法です」「絶対に大丈夫です」「必ず補償します」
- 相手の過失を決めつける言い方（例: 「そちらが悪いので」）
- 過度に専門用語だけで押し切る説明（用語が出たら `docs/terms/README.md` を参照・追記）

## 回答テンプレ（基本形）

状況に応じて、必要なブロックだけを使います（全部を毎回入れない）。

1. **結論（1〜2行）**
2. **重要ポイント（最大3点）**（該当する場合）
3. **手順案内**（順番が重要な場合）
4. **確認したいこと（質問）**（回答に必要な情報が不足している場合）
5. **次のサポート提案**（必要そうな場合のみ）
6. **参照先（リンク）**（根拠文書を1〜2つまで）

実際の雛形は `templates/` を参照してください。

- `templates/answer-template.md`
- `templates/escalation-template.md`

## エスカレーション（即対応の切り替え）

次に該当する場合は、通常回答を続けず、社内で確認・エスカレーションします。

- **法令**、**警察**、**報道**
- **炎上兆候**（SNS拡散、脅迫的文面、集団的問い合わせ等）
- **個人情報**（本人確認情報の取り扱い、漏えいの懸念）
- **規約解釈の争い**
- **詐欺の疑い**

エスカレーション時の返信雛形は `templates/escalation-template.md` を参照してください。

## Q&Aアーカイブ（校正の前提）

やり取りは `archive/` に記録し、後から誤回答の指摘・修正ができるようにします。

- **保存先**: `docs/support/line-qa/archive/`
- **形式**: 1件 = 1ファイル（Markdown）
- **テンプレ**: `templates/archive-entry-template.md`
