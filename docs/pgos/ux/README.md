# PGOS UXデザイン

## 本書の位置づけ

本ディレクトリは、PGOS の各アプリケーションにおける画面フロー・ワイヤーフレーム・ナビゲーション構造を定義するドキュメント群です。要件定義（`docs/pgos/requirements/`）で定めた機能要件を、具体的な画面・操作として落とし込みます。

---

## UX設計原則

### 1. モバイルファースト

- プレイヤー向け機能はスマートフォンでの操作を最優先に設計する
- 親指が届くゾーン（Thumb Zone）を意識したボタン・ナビゲーション配置
- 店舗向け機能はタブレット横置き操作を前提とし、片手でも操作できるレイアウト

### 2. 最少ステップの原則

- 主要タスクは **3タップ以内** で開始できること
  - 「今日ポーカーがしたい」→ ホーム → 店舗選択 → ウェイティング登録
  - 「トーナメントを始める」→ ダッシュボード → トーナメント選択 → 開始
- 繰り返し操作（チップイン/アウト、脱落記録等）は **1タップ** で完了するUI

### 3. リアルタイム性

- テーブル状況・ウェイティング人数・トーナメント進行はリアルタイム更新（ポーリングではなく WebSocket）
- 変更があった箇所を視覚的にハイライト（数値のカウントアップアニメーション等）
- プッシュ通知は即時性が必要なもの（順番到来、トーナメント開始等）に限定し、通知疲れを防ぐ

### 4. コンテキスト認識

- 現在地・時間帯に基づくインテリジェントな表示
  - 近くの店舗を優先表示
  - 営業中の店舗を上位に
  - 開始直前のイベントをハイライト
- チェックイン中は「現在プレイ中」の状態を反映した画面構成に切り替え

### 5. プログレッシブ・ディスクロージャー

- 初めてのユーザーには最小限の情報を表示し、詳細は展開式で提供
- 上級者向け設定（ストラクチャーの細かい設定等）は「詳細設定」として折りたたむ
- オンボーディングは段階的に（登録時に全情報を求めない）

### 6. エラー防止と復旧

- 破壊的操作（トーナメント削除、ポイント付与等）は確認ダイアログを表示
- 入力ミスを防ぐバリデーション（リアルタイムフィードバック）
- オフライン時も中断なく操作を続行でき、復帰時に自動同期（特にトーナメントクロック）

### 7. アクセシビリティ

- WCAG 2.1 AA 準拠（コントラスト比 4.5:1 以上、フォーカス表示、スクリーンリーダー対応）
- ダークモード対応（トーナメントクロックは暗い店舗環境で視認性を確保）
- 最小タッチターゲット 44x44pt（iOS）/ 48x48dp（Android）

## ナビゲーションパターン

### PGOS Player（モバイルアプリ）

**ボトムタブ + スタック型ナビゲーション**

```
┌─────────────────────────────────┐
│          画面コンテンツ           │
│                                 │
│                                 │
├─────┬─────┬─────┬─────┬─────────┤
│ホーム│検索  │ My  │通知  │メニュー  │
└─────┴─────┴─────┴─────┴─────────┘
```

- ボトムタブ5つで主要セクションを切り替え
- 各タブ内はスタック式に画面を積む（戻るボタンで1つ前に戻る）
- モーダル表示はチェックイン・eKYC・ポイント利用など一時的なフロー

### PGOS Manager（店舗管理Webアプリ）

**サイドバー + コンテンツエリア**

```
┌────────┬──────────────────────────┐
│        │ ヘッダー（店舗名・通知）    │
│ サイド  ├──────────────────────────┤
│ バー   │                          │
│        │      コンテンツエリア       │
│ ・Dashboard                       │
│ ・Tournament                      │
│ ・Ring Game                       │
│ ・Members   │                     │
│ ・Staff     │                     │
│ ・Points    │                     │
│ ・Reports   │                     │
│ ・Settings  │                     │
└────────┴──────────────────────────┘
```

- 左サイドバーで機能セクションを切り替え
- サイドバーは折りたたみ可能（タブレット時）
- コンテンツエリアは一覧 → 詳細のマスター/ディテール構成

### PGOS Clock（トーナメントタイマー）

**フルスクリーン固定レイアウト**

- 常にフルスクリーン表示
- 操作は別デバイス（PGOS Manager）から行い、Clock自体はディスプレイ専用
- 情報密度を最小限にし、遠くからでも読める大きなフォント

### PGOS Portal（公開Webサイト）

**トップナビゲーション + レスポンシブ**

- ヘッダーに主要ナビゲーション（店舗検索 / イベント / ランキング）
- モバイルではハンバーガーメニューに収納
- フッターに補足リンク（About / FAQ / 利用規約等）

## 競合UIの参照ポイント

| 競合 | 取り込むべき良い点 | PGOSでの改善方向 |
|------|-------------------|-----------------|
| **Poker Fans** | シンクロタイマーのマルチデバイス体験、無料プランでの基本UI | タイマー画面のブランディングカスタマイズをさらに強化 |
| **Waitinglist** | リアルタイム来店表示、チップ増減の自動記録UI、飲食注文の手軽さ | トーナメント管理も同等のリアルタイム体験に |
| **Hendon Mob** | プレイヤープロフィールの情報密度、多角的ランキング | モバイルネイティブで同等の閲覧体験を提供 |
| **Poker Timer Pro** | シンプルで見やすいクロックUI | カスタムブランディング + 参加者情報のオーバーレイ |

## 画面IDの命名規則

各画面にはユニークなIDを付与する:

| プレフィックス | アプリ | 例 |
|-------------|-------|-----|
| `P-` | PGOS Player | `P-HOME`, `P-STORE-DETAIL` |
| `M-` | PGOS Manager | `M-DASH`, `M-TN-LIST` |
| `C-` | PGOS Clock | `C-MAIN`, `C-BREAK` |
| `W-` | PGOS Portal | `W-TOP`, `W-STORE-DIR` |
