# PGOS 要件定義 — プラットフォーム・非機能要件

## 概要

本書は PGOS のアプリケーション構成、技術基盤、非機能要件（パフォーマンス・セキュリティ・可用性等）を定義する。

---

## C1. マルチプラットフォーム対応

### C1-1. アプリケーション構成


| コンポーネント             | 対象ユーザー              | 形態                                | 優先度    |
| ------------------- | ------------------- | --------------------------------- | ------ |
| **PGOS Player**     | プレイヤー               | iOS / Android ネイティブアプリ            | Must   |
| **PGOS Player Web** | プレイヤー               | PWA (Progressive Web App)         | Should |
| **PGOS Manager**    | 店舗オペレーター            | Web アプリ（レスポンシブ、タブレット最適化）          | Must   |
| **PGOS Floor**      | 店舗スタッフ（フロア / ディーラー） | iOS / Android ネイティブアプリ（ブラウザで代替可能） | Should |
| **PGOS Clock**      | 店舗オペレーター            | スマートTV / Fire TV / ブラウザ向け         | Must   |
| **PGOS Admin**      | PGOS管理者             | Web ダッシュボード                       | Must   |
| **PGOS Portal**     | 一般公開                | Web サイト（SSR/SSG）                  | Must   |
| **PGOS API**        | 外部連携                | RESTful API / GraphQL             | Must   |


### C1-2. デバイス・OS対応


| プラットフォーム   | 対応範囲                             | 優先度    |
| ---------- | -------------------------------- | ------ |
| iOS        | 16 以降                            | Must   |
| Android    | 10 以降                            | Must   |
| Web ブラウザ   | Chrome / Safari / Edge（最新2バージョン） | Must   |
| Fire TV    | Fire OS 7 以降                     | Must   |
| Android TV | Android TV 10 以降                 | Should |
| Apple TV   | tvOS 16 以降                       | Could  |


### C1-3. レスポンシブ設計


| ID      | 要件                                                    | 優先度  |
| ------- | ----------------------------------------------------- | ---- |
| C1-3-01 | PGOS Manager はデスクトップ（1280px以上）とタブレット（768px以上）の両方で最適表示 | Must |
| C1-3-02 | PGOS Portal はモバイル・タブレット・デスクトップの全サイズに対応                | Must |
| C1-3-03 | PGOS Clock はフルスクリーン表示に最適化（アスペクト比 16:9 / 16:10）        | Must |

## C2. グローバル対応

### C2-1. 多言語 (i18n)


| ID      | 要件                                            | 優先度    |
| ------- | --------------------------------------------- | ------ |
| C2-1-01 | 全UIテキストは翻訳キー経由で参照し、ハードコードしない                  | Must   |
| C2-1-02 | 初期対応言語: 日本語 / 英語 / ポルトガル語                     | Must   |
| C2-1-03 | 言語追加は設定ファイルの追加のみで対応できる設計                      | Must   |
| C2-1-04 | ユーザーごとの言語設定（ブラウザ/OS設定のフォールバック）                | Must   |
| C2-1-05 | 管理画面（PGOS Manager）のコンテンツ入力時に多言語対応（店舗名・イベント名等） | Should |
| C2-1-06 | RTL（右から左）言語への対応準備（レイアウトの方向を抽象化）               | Could  |


### C2-2. 多通貨


| ID      | 要件                                   | 優先度  |
| ------- | ------------------------------------ | ---- |
| C2-2-01 | 通貨はクラブ（店舗）単位で設定                      | Must |
| C2-2-02 | 初期対応通貨: JPY / USD / BRL / EUR        | Must |
| C2-2-03 | ポイントシステムは通貨と独立した単位として管理              | Must |
| C2-2-04 | 金額表示のフォーマット（桁区切り・小数点・通貨記号位置）はロケールに準拠 | Must |


### C2-3. タイムゾーン


| ID      | 要件                                               | 優先度  |
| ------- | ------------------------------------------------ | ---- |
| C2-3-01 | 全日時データは UTC で保存し、表示時にユーザーのタイムゾーンに変換              | Must |
| C2-3-02 | クラブ（店舗）ごとにデフォルトタイムゾーンを設定可能                       | Must |
| C2-3-03 | イベント・スケジュールの表示はユーザーのタイムゾーンで表示（オプションで店舗タイムゾーンも表示） | Must |

## C4. セキュリティ・信頼性

### C4-1. 認証・認可


| ID      | 要件                                       | 優先度    |
| ------- | ---------------------------------------- | ------ |
| C4-1-01 | メールアドレス + パスワードによる認証                     | Must   |
| C4-1-02 | ソーシャルログイン（Google / Apple / LINE）         | Should |
| C4-1-03 | 二要素認証 (2FA)（管理者・オペレーター向け）                | Must   |
| C4-1-04 | RBAC (Role-Based Access Control) による権限管理 | Must   |
| C4-1-05 | セッション管理（有効期限・強制ログアウト）                    | Must   |
| C4-1-06 | パスワードリセット機能                              | Must   |


### C4-2. データ保護


| ID      | 要件                                                         | 優先度  |
| ------- | ---------------------------------------------------------- | ---- |
| C4-2-01 | 通信の暗号化: TLS 1.2 以上を必須                                      | Must |
| C4-2-02 | 保存データの暗号化: 個人情報・決済情報は AES-256 で暗号化                         | Must |
| C4-2-03 | パスワードはハッシュ化（bcrypt / Argon2）して保存。正式登録時にメール+パスワードで登録する場合に適用 | Must |
| C4-2-04 | eKYC 関連データ（身分証画像等）の安全な保管・アクセス制限                            | Must |
| C4-2-05 | データベースのバックアップ（日次自動バックアップ、地理的冗長）                            | Must |
| C4-2-06 | 個人情報の削除リクエスト対応（GDPR / 個人情報保護法の「消去権」）                       | Must |


### C4-3. プライバシー


| ID      | 要件                         | 優先度    |
| ------- | -------------------------- | ------ |
| C4-3-01 | GDPR 対応（EU一般データ保護規則）       | Should |
| C4-3-02 | 個人情報保護法（日本）対応              | Must   |
| C4-3-03 | プライバシーポリシーへの同意フロー          | Must   |
| C4-3-04 | Cookie 同意バナー（Web）          | Must   |
| C4-3-05 | データポータビリティ（ユーザーデータのエクスポート） | Should |


### C4-3A. 決済基盤


| ID       | 要件                                                             | 優先度    |
| -------- | -------------------------------------------------------------- | ------ |
| C4-3A-01 | オンライン決済はクラブの銀行口座に直接入金される設計（プラットフォーム型決済。Stripe Connect 等）       | Must   |
| C4-3A-02 | 当社はプラットフォーム手数料のみを収受し、売上金を預からない構造                               | Must   |
| C4-3A-03 | クラブごとの決済アカウント（Stripe Connect Express / Standard 等）のオンボーディングフロー | Must   |
| C4-3A-04 | 決済消し込み方式の対応: キャンセル時に「返金ポイント付与」と「決済自体の取消し」を選択可能                 | Should |
| C4-3A-05 | 後払い（退店時一括）の未精算残高管理・督促機能                                        | Must   |
| C4-3A-06 | PCI DSS 準拠（カード情報は決済代行側で管理、自社サーバに保持しない）                         | Must   |


**決済フローの設計方針:**

- 売上金が当社を経由しない構造とすることで、資金移動業の規制回避とトラブル防止を実現
- クラブ側の PGOS Manager で決済アカウントのセットアップを行い、KYB（法人本人確認）を完了させる
- 手数料体系: 決済金額に対するプラットフォーム手数料（%）を当社が受け取る形

### C4-4. 監査・ログ

| ------- | ----------------------------- | ------ |

| C4-4-04 | ログの保存期間: 最低5年                 | Must   |
| C4-4-05 | ログのリアルタイムモニタリング・アラート          | Should |

| ID      | 要件                              | 優先度    |
| ------- | ------------------------------- | ------ |
| C4-5-01 | SLA: 99.9% の可用性（月間ダウンタイム 43分以内） | Must   |
| C4-5-02 | API レスポンスタイム: p95 で 500ms 以内    | Must   |
| C4-5-03 | 水平スケーリング対応（トラフィック増に応じた自動スケール）   | Must   |
| C4-5-04 | CDN によるスタティックコンテンツ配信            | Must   |
| C4-5-05 | データベースの読み取りレプリカによる負荷分散          | Should |
| C4-5-06 | 大規模大会時（同時接続1万人以上）の負荷テスト実施       | Should |


### C4-6. 障害対応


| ID      | 要件                                     | 優先度    |
| ------- | -------------------------------------- | ------ |
| C4-6-01 | 障害検知の自動アラート（Slack / メール / PagerDuty 等） | Must   |
| C4-6-02 | 障害時の自動フェイルオーバー                         | Should |
| C4-6-03 | RTO (Recovery Time Objective): 1時間以内   | Must   |
| C4-6-04 | RPO (Recovery Point Objective): 1時間以内  | Must   |
