# PGOS 要件定義 — プレイヤー側 機能要件

## 対象コンポーネント

- **PGOS Player**（iOS / Android アプリ、PWA）
- **PGOS Portal**（Webサイト — 一部機能のみ）

---

## A1. 店舗・イベント発見

プレイヤーが「今日どこでポーカーを遊べるか」を素早く見つけられること。
現行の POKERGUILD スケジュールサイトと Waitinglist の店舗検索を統合・強化する。

### A1-1. 店舗検索

| ID | 要件 | 優先度 |
|----|------|--------|
| A1-1-01 | 地図ベースで店舗を検索できる（現在地周辺・エリア指定） | Must |
| A1-1-02 | 条件フィルタ: ゲーム種別（NLH / PLO / ミックス等）、レート帯、営業時間 | Must |
| A1-1-03 | 店舗の営業ステータス（営業中 / 準備中 / 休業）をリアルタイム表示 | Should |
| A1-1-04 | 店舗詳細ページ: 住所・アクセス・営業時間・写真・設備情報・SNSリンク | Must |
| A1-1-05 | 店舗評価・レビュー機能 | Could |
| A1-1-06 | 検索履歴・最近訪問した店舗の表示 | Should |

### A1-2. イベント・スケジュール検索

| ID | 要件 | 優先度 |
|----|------|--------|
| A1-2-01 | カレンダー/リスト形式でトーナメント・イベントを一覧表示 | Must |
| A1-2-02 | 日付・エリア・ゲーム種別・エントリー費用帯でフィルタ | Must |
| A1-2-03 | トーナメント詳細: ストラクチャー・エントリー費・ギャランティー・開始時刻 | Must |
| A1-2-04 | リングゲームの開催状況（テーブル数・レート・待ち人数）をリアルタイム表示 | Should |
| A1-2-05 | 大規模大会（フェスティバル/シリーズ）のまとまった情報ページ | Should |
| A1-2-06 | iCal / Google Calendar 連携（イベントの外部カレンダー登録） | Could |

### A1-3. お気に入り・通知

| ID | 要件 | 優先度 |
|----|------|--------|
| A1-3-01 | 店舗をお気に入り登録し、新しいイベント情報をプッシュ通知で受信 | Must |
| A1-3-02 | 特定の条件（エリア・レート帯等）に合うイベントが登録されたときの自動通知 | Should |
| A1-3-03 | フォロー中のプレイヤーがイベントに登録したときの通知 | Could |

### A2-3. トーナメント参加

## A3. 戦績・ランキング

プレイヤーの実績を記録・可視化し、Hendon Mob / GPI に対抗するデータベースを構築する。

| ID | 要件 | 優先度 |
|----|------|--------|

| A3-1-02 | 戦績サマリーの表示（トーナメント参加数・入賞数・獲得ポイント累計等） | Must |
| A3-1-03 | プロフィールの公開/非公開設定 | Must |
| A3-1-04 | Hendon Mob / GPI プロフィールへのリンク設定 | Could |
| A3-1-05 | SNS アカウント連携表示 | Could |

### A3-2. トーナメント戦績

| ID | 要件 | 優先度 |

| A3-2-02 | トーナメント戦績の一覧表示（時系列・フィルタ可能） | Must |
| A3-2-03 | 獲得ポイント推移のグラフ表示 | Should |
| A3-2-04 | トーナメント種別別（NLH / PLO / Mix 等）の成績集計 | Should |

### A3-3. リングゲーム戦績

| ID | 要件 | 優先度 |
|----|------|--------|
| A3-3-01 | セッション（着席〜離席）ごとのプレイ時間を自動記録 | Must |
| A3-3-02 | 総プレイ時間・セッション数・訪問店舗数のサマリー表示 | Must |
| A3-3-03 | レート別・店舗別のプレイ時間集計 | Should |

### A3-4. ランキング

| ID | 要件 | 優先度 |
|----|------|--------|
| A3-4-01 | トーナメント獲得ポイントランキング（月別・年別・累計） | Must |

|----|------|--------|

| A3-5-06 | 仮会員も経験値を蓄積可能。正式登録後に仮会員データとマージして引き継ぐ | Should |

## A6. 本人確認 (eKYC)

ポイントシステムの利用やセキュリティ確保のため、プレイヤーの本人確認を行う。
現行 GameID の eKYC を発展させる。

### A6-1. 本人確認手続き

| ID | 要件 | 優先度 |
|----|------|--------|
| A6-1-01 | アプリ内で身分証明書（パスポート/運転免許証/マイナンバーカード）の撮影・提出 | Must |
| A6-1-02 | 顔写真の撮影と身分証写真との照合（ライブネス検出） | Must |
| A6-1-03 | 海外身分証明書への対応（グローバル展開前提） | Should |
| A6-1-04 | 審査結果のプッシュ通知・ステータス確認 | Must |
| A6-1-05 | 審査は原則1営業日以内に完了 | Must |
| A6-1-06 | IC認証（NFC）方式の対応: パスポート/運転免許証/マイナンバーカード/在留カードのICチップ読取 | Should |
| A6-1-07 | パスポートOCR: MRZ（機械読取領域）からの自動入力（氏名・国籍・生年月日等） | Should |
| A6-1-08 | 身分証の有効期限管理: 身分証番号・有効期限をテキストデータとして保存（画像は審査完了14日後に自動削除）。期限切れ時に再認証を要求 | Should |
| A6-1-09 | SMS認証（電話番号認証）: 正式登録後にSMS認証を追加可能 | Should |

### A6-2. 年齢確認

| ID | 要件 | 優先度 |
|----|------|--------|
| A6-2-01 | 18歳未満のプレイヤーはポイント関連機能を利用不可 | Must |
| A6-2-02 | 年齢確認は eKYC プロセスの一部として実施 | Must |
| A6-2-03 | 年代タグの自動生成（生年月日から算出）: 会員QR付近に表示しスタッフが目視確認可能 | Must |
| A6-2-04 | 青少年育成条例対応: 都道府県ごとの時間制限設定をクラブが管理（18歳未満/16歳未満、同伴有無による分岐） | Should |
| A6-2-05 | ペアレンタルコントロール: 保護者が未成年のプレイ時間・エントリー制限を管理 | Could |

### A6-3. 会員登録の段階的拡張

| ID | 要件 | 優先度 |
|----|------|--------|
| A6-3-01 | 仮会員登録: ユーザーID（英数字ユニーク）+ 生年月日のみで会員QRを即時発行 | Must |
| A6-3-02 | 正式登録: メール認証（ワンタイムURL）またはソーシャルログイン + 国の入力。仮会員データのマージ | Must |
| A6-3-03 | eKYC: 正式登録後に任意で実施。ポイント利用・トランスファー等に必要 | Must |
| A6-3-04 | ソーシャルログイン対応: Google / Apple / LINE / Facebook / X / Yahoo | Must |
| A6-3-05 | 2FA（オプトイン）: TOTP アプリまたはSMS | Should |
| A6-3-06 | 生体認証: Face ID / Touch ID による認証（2回目以降のログイン + 決済確認） | Should |
| A6-3-07 | 紹介特典制度: 招待コードによる紹介 → 紹介者・被紹介者双方に特典付与 | Could |

## ユーザーストーリー（主要シナリオ）

### シナリオ 1: 「今日ポーカーがしたい」

> プレイヤーA は仕事帰りにポーカーを遊びたい。PGOS Player アプリを開き、現在地周辺の店舗を検索。NLH 1-2 のリングゲームが開催中の店舗を見つけ、ウェイティングリストに登録。電車で移動中に「あと2人です」の通知を受信。店舗到着後、QRコードでチェックイン。プレイ終了後、プレイ時間が自動記録される。

### シナリオ 2: 「週末のトーナメントに参加したい」

> プレイヤーB はカレンダーで週末のトーナメントを検索。エントリー費 ¥5,000 + ウェブコイン利用可能なイベントを発見。アプリから事前エントリーし、ポイントで支払い。当日会場でチェックインし、トーナメントに参加。3位入賞し、選手契約済みのためポイントが自動付与される。結果はプロフィールの戦績に反映。

### シナリオ 3: 「自分のランキングを確認したい」

> プレイヤーC はプロフィールページを開き、今月のトーナメント成績を確認。地域別ランキングで現在15位であることを確認。フレンドのプレイヤーD が同じイベントで入賞したことをフィードで知り、「いいね」を送る。

## 非機能要件（プレイヤー側固有）

| 項目 | 要件 |
|------|------|
| アプリ起動時間 | 3秒以内（コールドスタート） |
| 検索レスポンス | 店舗/イベント検索結果の表示まで 1秒以内 |
| リアルタイム更新 | ウェイティングリスト・テーブル状況の更新は 5秒以内に反映 |
| オフライン対応 | 直近の検索結果・プロフィール・戦績はオフラインでも閲覧可能 |
| アクセシビリティ | WCAG 2.1 AA 準拠 |
| 対応OS | iOS 16以降、Android 10以降 |
