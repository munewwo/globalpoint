# PGOS 要件定義 — フェーズ分け・ロードマップ

## 概要

PGOS の開発を4つのフェーズに分け、段階的にリリースする。各フェーズは独立してリリース可能な価値を持ち、フィードバックを反映しながら段階的に機能を拡張する。

---

## フェーズ全体像

```
2026                                      2027
Q2         Q3         Q4         Q1         Q2         Q3
├──────────┼──────────┼──────────┼──────────┼──────────┤
│◀── Phase 1 (MVP) ──▶│                                │
│                      │◀── Phase 2 ──▶│               │
│                                       │◀─ Phase 3 ─▶│
│                                       │◀── Phase 4 ──▶│
```

| フェーズ | 期間（目安） | テーマ | 目標 |
|---------|------------|--------|------|
| Phase 1 | 2026 Q2〜Q3 (6ヶ月) | MVP — 店舗運営の核とプレイヤー基盤 | 最小限の機能で数店舗でパイロット運用開始 |
| Phase 2 | 2026 Q4〜2027 Q1 (4ヶ月) | 拡張 — ポイントシステムと戦績DB | ウェブコイン後継の本格運用、Hendon Mob対抗の戦績データベース |
| Phase 3 | 2027 Q1〜Q2 (3ヶ月) | 差別化 — ソーシャルと高度分析 | 競合にないユーザー体験、経営分析機能 |
| Phase 4 | 2027 Q2〜Q3 (4ヶ月) | 移行・展開 — 既存サービス移行とグローバル展開 | 全クラブの移行完了、海外市場への展開 |

---

## Phase 1: MVP（2026 Q2〜Q3）

### 目標

- ポーカー店舗がトーナメント・リングゲームを運営できる最小限の機能を提供
- プレイヤーが店舗を探してイベントに参加できる基盤を構築
- 数店舗でのパイロット運用を開始し、フィードバックを収集

### 対象機能

| 機能領域 | 対象 ID | Must 項目数 | 概要 |
|---------|---------|------------|------|
| **B1. トーナメント管理** | B1-1〜B1-5 | 20+ | ストラクチャー、クロック（マルチデバイス同期・ショットクロック）、参加者管理、テーブル管理、ペイアウト |
| **B2. リングゲーム管理** | B2-1〜B2-3 | 10+ | テーブル管理、ウェイティングリスト、セッション管理 |
| **B3. 会員管理** | B3-1〜B3-2 | 8+ | 会員登録（QR）、会員情報管理 |
| **B6. スタッフ管理** | B6-1〜B6-3 | 6+ | アカウント管理、権限管理、監査ログ |
| **A1. 店舗・イベント発見** | A1-1〜A1-3 | 6+ | 店舗検索（マップ）、スケジュール検索、お気に入り |
| **A2. プレイ体験** | A2-1〜A2-3 | 8+ | ウェイティングリスト、チェックイン、トーナメント参加、ショットクロック（チェスクロック方式） |
| **A6. 本人確認 (eKYC)** | A6-1〜A6-2 | 5+ | 身分証撮影、顔照合、年齢確認 |
| **C1〜C4 基盤** | — | — | API基盤、認証、マルチ言語基盤、セキュリティ |

### 成果物

- PGOS Manager (Web) — トーナメント管理・リングゲーム管理・会員管理
- PGOS Clock (Web/TV) — マルチデバイス同期タイマー
- PGOS Player (iOS/Android) — 店舗検索・ウェイティング・チェックイン・eKYC
- PGOS Portal (Web) — 店舗・イベント検索ページ
- PGOS API — 内部 API 基盤

### マイルストーン

| 時期 | マイルストーン |
|------|-------------|
| 月1〜2 | 技術スタック確定、アーキテクチャ設計、DB設計 |
| 月2〜3 | PGOS Manager / Clock のプロトタイプ |
| 月3〜4 | PGOS Player のプロトタイプ |
| 月4〜5 | 統合テスト、パイロット店舗の選定・準備 |
| 月5〜6 | パイロット運用開始（5〜10店舗）、フィードバック収集 |

## Phase 2: 拡張（2026 Q4〜2027 Q1）

### 目標

- ウェブコイン後継のポイントシステムを本格稼働
- Hendon Mob に対抗する戦績データベース・ランキングを構築
- パイロット店舗の範囲を拡大

### 対象機能

| 機能領域 | 対象 ID | 概要 |
|---------|---------|------|
| **A4. ポイント・報酬** | A4-1〜A4-3 | ポイント残高・利用・選手契約・大会補助 |
| **B5. ポイントシステム運営** | B5-1〜B5-3 | 付与リクエスト・利用受付・コンプライアンスチェック |
| **A3. 戦績・ランキング** | A3-1〜A3-4 | プレイヤープロフィール・トーナメント/リング戦績・ランキング |
| **B7. 店舗プレゼンス** | B7-1〜B7-3 | 店舗ページ・スケジュール公開・マーケティング支援 |
| **B4. 売上・会計** | B4-1〜B4-4 | POS機能・レポート・ポイント精算・広告料管理 |
| **ポイントシステム詳細** | WC-1〜WC-8 | 付与枠・付与フロー・認定制度・利用・トランスファー・精算・選手契約・大会補助 |

### 成果物

- PGOS ポイントシステム — 付与・利用・精算の全フロー
- 戦績データベース — トーナメント/リングゲーム結果の蓄積・表示
- ランキングシステム — 複数軸のランキング（PGOS Portal で公開）
- 売上レポート — 店舗向けの経営分析機能
- PGOS Admin (Web) — コンプライアンス管理ダッシュボード

### マイルストーン

| 時期 | マイルストーン |
|------|-------------|
| 月1〜2 | ポイントシステムのデータモデル・フロー設計、コンプライアンスレビュー |
| 月2〜3 | ポイント付与・利用フローの実装、戦績DB構築 |
| 月3 | ポイントシステムのパイロット運用（ウェブコイン利用店舗の一部） |
| 月3〜4 | ランキング・レポート機能の実装、店舗プレゼンス機能 |
| 月4 | Phase 2 機能のリリース、利用店舗の拡大 |

## Phase 3: 差別化（2027 Q1〜Q2）

### 目標

- 競合にないソーシャル機能でプレイヤーのエンゲージメントを向上
- 高度なアナリティクスで店舗経営を支援
- PGOS をポーカーコミュニティのプラットフォームとして確立

### 対象機能

| 機能領域 | 対象 ID | 概要 |
|---------|---------|------|
| **A5. ソーシャル** | A5-1〜A5-3 | フレンド機能・メッセージ・アクティビティフィード |
| **A2-4. 飲食オーダー** | A2-4-01〜03 | アプリ内飲食注文 |
| **B4 高度化** | B4-2-07 等 | トレンド分析・前年比較・予測 |
| **B3-3〜B3-4** | — | 来店分析高度化・コミュニケーション強化 |
| **B1-6** | B1-6-03〜04 | シリーズ/フェスティバル管理・通算ランキング |

### 成果物

- ソーシャル機能（フレンド・来店状況・アクティビティフィード）
- 飲食オーダーシステム
- 高度アナリティクスダッシュボード
- Public API / Webhook
- シリーズ/フェスティバル管理

### マイルストーン

| 時期 | マイルストーン |
|------|-------------|
| 月1 | ソーシャル機能設計、飲食オーダーの業務フロー整理 |
| 月1〜2 | フレンド機能・来店状況表示の実装 |
| 月2〜3 | 飲食オーダー・高度アナリティクスの実装 |

## Phase 4: 移行・展開（2027 Q2〜Q3）

### 目標

- 既存 POKERWEB / GameID / POKERGUILD からの完全移行
- 海外市場への本格展開

### 対象機能

| 機能領域 | 概要 |
|---------|------|
| **データ移行** | クラブ・プレイヤー・ウェブコイン・戦績データの移行 |
| **並行運用** | POKERWEB と PGOS の並行運用・段階的切替 |
| **多言語展開** | 英語・ポルトガル語の本格対応、追加言語への拡張 |
| **海外クラブ対応** | 海外店舗の登録・運用開始 |
| **旧サービス停止** | POKERWEB / GameID の段階的停止 |

### マイルストーン（[migration.md](migration.md) と連動）

| 時期 | マイルストーン |
|------|-------------|
| 月1 | 移行ツール完成、テスト環境での移行リハーサル |
| 月1〜2 | パイロット移行（5〜10クラブ） |
| 月2〜3 | 段階移行（残りクラブを順次移行） |
| 月3〜4 | 全クラブ移行完了確認、旧サービス停止準備 |
| 月4 | 旧サービス停止、PGOS 完全移行完了 |

## 料金プラン案

競合（特に Poker Fans の無料プラン）への対抗と、現行 POKERWEB の料金体系を踏まえた PGOS の料金プラン案。

| プラン | 月額（税込） | 主な制限 | 対象 |
|--------|------------|---------|------|
| **Free** | ¥0 | トーナメント月10回、会員100人、タイマー1台、ポイント非対応 | 小規模店舗・試用 |
| **Standard** | ¥9,900 | トーナメント月40回、会員1,000人、タイマー3台、ポイント対応 | 標準的な店舗 |
| **Professional** | ¥19,800 | トーナメント月100回、会員5,000人、タイマー10台、高度レポート | 中〜大規模店舗 |
| **Enterprise** | ¥33,000 | 無制限、API利用、カスタムブランディング、優先サポート | 大規模店舗・チェーン |
| **Event** | 個別見積 | 大規模大会・フェスティバル向けの一時利用 | イベント主催者 |

**ポイントシステム**: Standard 以上のプランで利用可能。広告料は別途。

## 前提条件・リスク

### 前提条件

| 項目 | 内容 |
|------|------|
| 開発チーム | フロントエンド 2〜3名、バックエンド 2〜3名、モバイル 1〜2名、デザイナー 1名、QA 1名（Phase 1 時点） |
| インフラ | クラウドプラットフォーム（AWS / GCP）を使用 |
| ブラジル開発元 | データエクスポートへの協力が得られること |
| 法務 | 顧問弁護士によるポイントシステムの法的レビューが完了していること |

### 主要リスク

| リスク | 影響度 | 発生可能性 | 対策 |
|--------|--------|-----------|------|
| ブラジル開発元のデータエクスポート協力が得られない | 高 | 中 | 早期の協議開始、代替データ取得方法の検討 |
| ポイントシステムの法的リスク | 高 | 低 | 顧問弁護士の早期レビュー、段階的なコンプライアンス確認 |
| 競合の急速な機能拡充 | 中 | 高 | MVP の早期投入、差別化ポイント（ポイントシステム）への集中 |
| パイロット店舗での予想外の運用課題 | 中 | 高 | パイロット期間の十分な確保、迅速なフィードバックループ |
| 開発リソース不足 | 高 | 中 | 優先度の厳格な管理、Phase 1 のスコープ絞り込み |
| 既存ユーザーの移行抵抗 | 中 | 中 | 並行運用期間の確保、手厚い移行サポート |

## 次のステップ（要件定義完了後）

1. **技術スタック選定**: フロントエンド / バックエンド / モバイル / インフラの技術選定
2. **UI/UX デザイン**: ワイヤーフレーム → プロトタイプの作成
3. **DB 設計**: データモデルの詳細設計
4. **API 設計**: エンドポイント仕様の策定
5. **パイロット店舗の選定**: Phase 1 MVP の検証パートナーの確保
6. **ブラジル開発元との協議**: データ移行に関する技術的・契約的な合意
7. **法務レビュー**: ポイントシステム設計のコンプライアンス確認
