# PGOS 要件定義 — 全体概要

## 本書の位置づけ

本書は PGOS (Poker Global Operating System) の要件定義全体を俯瞰するドキュメントです。
各機能領域の詳細は個別ドキュメントに記載し、本書では全体構造・設計方針・優先度の考え方を示します。

---

## 1. システム全体像

### 1.1 ユーザー区分

| 区分 | 説明 | 主な利用チャネル |
|------|------|----------------|
| **プレイヤー** | ポーカーをプレイする個人。店舗検索、イベント参加、戦績確認、ポイント管理を行う | モバイルアプリ (iOS/Android)、Webブラウザ |
| **店舗オペレーター** | ポーカー店舗のオーナー・スタッフ。トーナメント/リングゲーム運営、会員管理、売上管理を行う | Webアプリ、タブレットアプリ |
| **イベント主催者** | 大会・フェスティバルの主催者。複数日・複数会場のイベントを管理する | Webアプリ |
| **PGOS管理者** | グローバルポイント社の運営スタッフ。ポイントシステム運営、コンプライアンス管理、全体監視を行う | 管理用Webダッシュボード |

### 1.2 アプリケーション構成

| コンポーネント | 対象ユーザー | 形態 |
|---------------|-------------|------|
| **PGOS Player** | プレイヤー | iOS / Android ネイティブアプリ + PWA |
| **PGOS Manager** | 店舗オペレーター / イベント主催者 | Webアプリ（レスポンシブ）+ タブレット最適化 |
| **PGOS Floor** | 店舗スタッフ（フロア / ディーラー） | iOS / Android ネイティブアプリ。未導入時は PGOS Manager（ブラウザ）で全機能を代替可能。運用上なくても問題ないが、あると便利なツール |
| **PGOS Clock** | 店舗オペレーター | スマートTV / Fire TV / ブラウザ向けタイマー/クロック |
| **PGOS Admin** | PGOS管理者 | 管理用Webダッシュボード |
| **PGOS Portal** | 一般公開 | Webサイト（店舗・イベント検索、ランキング、スケジュール） |
| **PGOS API** | 外部開発者 / 連携先 | RESTful API / GraphQL |

**PGOS Floor の概要:**
- フロアスタッフ向けのモバイルアプリ。店舗内でのチェックイン処理、配席、バランシング操作等を行う
- **ディーラーモード**: テーブルQR + パスワードでログインし、指定テーブルに特化した表示に切り替わる。ニュープレイヤー通知、ショットクロック、バランシング対象者の選択等を行う
- マスト導入ではなく、未導入でもPGOS Manager（ブラウザ）ですべてのオペレーションが可能
- ディーラー端末の有無を前提としない設計とし、店舗が運用レベルに応じて選択できる

## 3. 設計方針

### 3.1 グローバルファースト

- 多言語対応（初期: 日本語・英語・ポルトガル語）は i18n フレームワークで設計
- 通貨・タイムゾーンはデータモデルレベルで対応
- 文字列はすべて翻訳キー経由で参照

### 3.2 モバイルファースト

- プレイヤー向け機能はモバイルアプリを主要チャネルとして設計
- 店舗向け機能はタブレット操作を前提としたUI設計
- オフライン時の最低限の動作保証（トーナメントクロック等）

### 3.3 API ドリブン

- すべてのフロントエンドは共通の API を通じてバックエンドとやり取り
- サードパーティ連携（外部トーナメントソフト、ランキングサイト等）を想定した API 設計
- Webhook によるイベント駆動連携

### 3.4 コンプライアンス・バイ・デザイン

- ポイントシステムの付与・利用・精算フローにコンプライアンスチェックを組み込み
- 操作ログ・監査証跡をすべての重要操作に対して記録
- 個人情報の取り扱いは GDPR / 個人情報保護法に準拠した設計

### 3.5 フリーミアム

- 基本機能（トーナメント管理、リングゲーム管理、会員管理）は無料で提供
- ポイントシステム、高度な分析、ブランディング機能を有料プランとして提供
- 競合の Poker Fans（無料プランあり）に対抗する導入障壁の低さ

## 4. 優先度の考え方（MoSCoW法）

各機能要件には以下の優先度を付与する:

| 優先度 | 意味 | Phase での位置づけ |
|--------|------|-------------------|
| **Must** | システムとして成立するために必須 | Phase 1 (MVP) |
| **Should** | 競合に対抗するために強く推奨 | Phase 2 (拡張) |
| **Could** | 差別化や利便性向上に貢献 | Phase 3 (差別化) |
| **Won't (now)** | 現時点では対象外だが将来検討 | Phase 4 以降 |

## 5. 用語定義

PGOS 固有の用語は以下のように定義する（既存の用語定義 `docs/terms/README.md` も参照）:

| 用語 | 定義 |
|------|------|
| **クラブ** | PGOS に登録されたポーカー店舗または大会主催団体の単位 |
| **オペレーター** | クラブに所属し、PGOS Manager を操作するスタッフ |
| **プレイヤー** | PGOS Player アプリに登録した個人 |
| **ストラクチャー** | トーナメントのブラインドレベル・時間配分を定めた構成表 |
| **ペイアウト** | トーナメントの賞金（またはポイント）配分表 |
| **セッション** | リングゲームにおけるプレイヤーの1回の着席〜離席の記録 |
| **PGOS ポイント** | ウェブコインの後継となるポイントシステム（仮称） |
| **eKYC** | 電子的な本人確認手続き |
