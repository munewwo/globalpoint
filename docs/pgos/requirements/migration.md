# PGOS 要件定義 — 移行要件

## 概要

本書は、現行サービス（POKERWEB / GameID / POKERGUILD）から PGOS への移行に関する要件を定義する。
約 300 以上のクラブと多数のプレイヤーが利用中のサービスを、データとサービス品質を維持しながら移行する計画を策定する。

---

## 1. 移行対象

### 1.1 現行サービスの利用規模（概算）

| サービス | 利用規模（2025年時点） |
|---------|---------------------|
| POKERWEB（店舗管理） | 247 クラブ |
| POKERWEB ウェブコイン利用 | 180 クラブ |
| POKERGUILD（スケジュール掲載） | 306 クラブ |
| GameID（プレイヤーアプリ） | プレイヤー数は非公開（推定数万人規模） |

### 1.2 移行対象データ

| データ種別 | 移行元 | 重要度 | 備考 |
|-----------|--------|--------|------|
| **クラブ（店舗）情報** | POKERWEB / POKERGUILD | 必須 | 店舗名・住所・連絡先・プラン情報等 |
| **プレイヤーアカウント** | GameID | 必須 | 氏名・メール・eKYC情報等 |
| **ウェブコイン残高** | POKERWEB | 必須 | 全プレイヤーの残高。1コインの誤差も許容しない |
| **ウェブコイン取引履歴** | POKERWEB | 必須 | 付与・利用・トランスファー・手数料の全記録 |
| **選手契約情報** | POKERGUILD | 必須 | 契約ステータス・締結日 |
| **トーナメント結果** | POKERWEB | 高 | 過去のトーナメント結果・ランキング |
| **リングゲームセッション** | POKERWEB | 高 | 過去のセッション記録 |
| **ストラクチャーテンプレート** | POKERWEB | 中 | 店舗が作成したテンプレート |
| **スケジュール・イベント情報** | POKERGUILD | 高 | 掲載中のイベント情報 |
| **eKYC 審査データ** | GameID | 必須 | 審査結果（身分証画像は方針に従い判断） |
| **スタッフアカウント** | POKERWEB | 高 | ログイン情報・権限設定 |
| **付与枠（広告料関連）** | POKERWEB | 必須 | 残高・有効期限 |

## 2. 移行方針

### 2.1 段階的移行

一斉切替（ビッグバン移行）ではなく、段階的に移行を実施する。

```
Phase A: 準備・データ移行基盤構築
     ↓
Phase B: パイロット移行（少数クラブ）
     ↓
Phase C: 段階的移行（クラブ群を順次移行）
     ↓
Phase D: 完全移行・旧サービス停止
```

### 2.2 並行運用期間

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-1-01 | 移行期間中は POKERWEB / GameID と PGOS の並行運用を行う | Must |
| MG-1-02 | 並行運用期間中、ウェブコイン残高は POKERWEB ⇔ PGOS 間で同期する | Must |
| MG-1-03 | 並行運用期間は最大6ヶ月を目安とする | Should |
| MG-1-04 | 並行運用中の新規クラブ登録は PGOS のみで受け付ける | Should |

## 3. データ移行要件

### 3.1 クラブ（店舗）データ

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-2-01 | POKERWEB の全クラブ情報を PGOS に移行する | Must |
| MG-2-02 | POKERGUILD のみに登録されているクラブ情報も PGOS に移行する | Must |
| MG-2-03 | 両方に登録されているクラブは名寄せ（重複排除）を行う | Must |
| MG-2-04 | プラン情報は PGOS の新料金体系にマッピングする | Must |
| MG-2-05 | 移行後のクラブ情報を店舗に確認してもらうフローを用意する | Should |

### 3.2 プレイヤーアカウント

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-3-01 | GameID の全プレイヤーアカウントを PGOS Player に移行する | Must |
| MG-3-02 | 認証情報（パスワードハッシュ）を移行し、パスワード再設定なしでログイン可能にする | Must |
| MG-3-03 | パスワードハッシュの移行が技術的に不可能な場合、初回ログイン時のパスワードリセットフローを用意する | Must |
| MG-3-04 | eKYC 審査済みステータスを引き継ぐ | Must |
| MG-3-05 | プレイヤーに移行の事前通知を行う（メール/アプリ通知） | Must |

### 3.3 ウェブコイン（ポイント）

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-4-01 | 全プレイヤーのウェブコイン残高を正確に PGOS ポイントに移行する | Must |
| MG-4-02 | 移行時のスナップショットを取得し、残高の整合性を検証する | Must |
| MG-4-03 | 移行前後の残高差異が0であることを自動検証する | Must |
| MG-4-04 | ウェブコイン取引履歴を PGOS のトランザクション履歴に移行する | Must |
| MG-4-05 | 付与枠の残高を PGOS に引き継ぐ | Must |
| MG-4-06 | 移行期間中のポイント取引の整合性を保証する仕組みを用意する | Must |

### 3.4 トーナメント・リングゲームデータ

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-5-01 | 過去のトーナメント結果データを PGOS に移行する（戦績データベース構築のため） | Must |
| MG-5-02 | リングゲームセッション記録を PGOS に移行する | Should |
| MG-5-03 | ストラクチャーテンプレートを PGOS に移行する | Should |
| MG-5-04 | ランキングデータは移行後に再計算する | Must |

### 3.5 スケジュール・イベント

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-6-01 | 移行時点で将来日付のイベント情報を PGOS に移行する | Must |
| MG-6-02 | 定期スケジュール（繰り返しイベント）の設定を移行する | Should |
| MG-6-03 | 過去のイベント情報はアーカイブとして保存する | Should |

| MG-7-03 | 外部サイトからの既存リンクが切れないよう対応する | Should |

### 4.2 GameID アプリの移行

| ID | 要件 | 優先度 |

| MG-8-03 | GameID アプリの段階的機能停止スケジュール | Must |

### 4.3 決済・契約

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-9-01 | POKERWEB の利用契約を PGOS の利用契約に切り替えるフロー | Must |
| MG-9-02 | 広告料契約の継続性を保証する | Must |
| MG-9-03 | 料金体系の変更がある場合、事前に十分な告知期間を設ける | Must |

## 6. ロールバック計画

移行に重大な問題が発生した場合に備え、ロールバック手順を定義する。

| ID | 要件 | 優先度 |
|----|------|--------|
| MG-12-01 | 移行前のデータの完全バックアップ | Must |
| MG-12-02 | 段階移行の各ステップでロールバック可能な設計 | Must |
| MG-12-03 | ロールバック判断基準の定義（残高不整合、重大障害等） | Must |
| MG-12-04 | ロールバック手順書の作成・訓練 | Must |

## 7. 移行スケジュール（概案）

| フェーズ | 期間（目安） | 内容 |
|---------|------------|------|
| **Phase A: 準備** | 2ヶ月 | データ移行ツール開発、移行手順書作成、テスト環境構築 |
| **Phase B: パイロット** | 1ヶ月 | 5〜10クラブで先行移行。問題点の洗い出しと修正 |
| **Phase C: 段階移行** | 3〜4ヶ月 | 残りのクラブを10〜30クラブ単位で順次移行 |
| **Phase D: 完全移行** | 1ヶ月 | 全クラブ移行完了確認、旧サービスの停止、リダイレクト設定 |

## 9. 成功基準

移行の成功を判断する基準を以下に定める。

| 基準 | 指標 |
|------|------|
| データ整合性 | ウェブコイン残高の移行前後差異が 0 |
| アカウント移行率 | 移行対象プレイヤーの 95% 以上が PGOS Player に移行 |
| クラブ移行率 | 移行対象クラブの 100% が PGOS Manager に移行 |
| サービス継続性 | 移行に伴うサービス停止時間が計画値以内（1クラブあたり最大2時間） |
| ユーザー満足度 | 移行後1ヶ月のサポート問い合わせ数が許容範囲内 |
