# PGOS 要件定義 — コンプライアンス要件

## 概要

PGOS は風営法・資金決済法・刑法等の法令を遵守し、安心して利用できるプラットフォームとして設計する。本書は、現行のコンプライアンスロードマップ（[docs/release/2025-06-05-compliance-roadmap.md](../../release/2025-06-05-compliance-roadmap.md)）の方針を踏襲しつつ、PGOS で対応すべき法的・運用的要件を定義する。

**重要**: 本書は法的助言ではなく、当社の運用方針をシステム要件として整理するものです。法令解釈・責任範囲に関しては顧問弁護士の確認を得て最終決定します。

---

## 1. 風営法遵守

### 1.1 基本方針

ポーカー店舗が風俗営業許可を取得して営業する前提において、風営法に抵触しないようシステム設計で支援する。

### 1.2 景品提供の回避


| ID      | 要件                                  | 実装方法                             | 優先度  |
| ------- | ----------------------------------- | -------------------------------- | ---- |
| CL-1-01 | ポイント付与は当社（グローバルポイント）が行い、店舗が直接付与できない | 付与フローに当社承認ステップを必須化               | Must |
| CL-1-02 | ポイント付与は選手契約（業務委託契約）に基づく報酬として位置づけ    | 契約未完了者への付与を自動ブロック                | Must |
| CL-1-03 | 店舗がプライズとして賞金を提供する構造にならないよう設計        | トーナメント結果とポイント付与を分離（リクエスト→承認の2段階） | Must |


### 1.3 遊技チップの管理


| ID      | 要件                          | 実装方法                                 | 優先度    |
| ------- | --------------------------- | ------------------------------------ | ------ |
| CL-2-01 | 遊技チップ＝ポイント（等価）の構造を禁止        | 認定制度の変換テーブルで等価チェック。等価率が閾値を超える場合にアラート | Must   |
| CL-2-02 | 「ウェブコインリング」等の等価を想起させる表記を禁止  | イベント名・説明文のNGワードチェック                  | Should |
| CL-2-03 | 遊技チップの店外持ち出しに該当しない料金システムの強制 | リングゲーム料金設定時のガイダンス表示                  | Must   |


### 1.4 高額サービスの自主規制


| ID      | 要件                                    | 実装方法          | 優先度    |
| ------- | ------------------------------------- | ------------- | ------ |
| CL-3-01 | トーナメントのエントリー費（アドオン含む）が税別10万円を超える設定を警告 | 金額入力時のバリデーション | Must   |
| CL-3-02 | リングゲームの料金が税別5万円を超える設定を警告              | 金額入力時のバリデーション | Must   |
| CL-3-03 | 後払い額が10万円を超える場合に精算を促す警告               | 後払い記録時の自動アラート | Must   |
| CL-3-04 | 上記規制値の管理画面からの変更機能（法改正・方針変更に対応）        | 管理者向け設定画面     | Should |


### 1.5 営業許可確認


| ID      | 要件                   | 実装方法            | 優先度    |
| ------- | -------------------- | --------------- | ------ |
| CL-4-01 | クラブ登録時に風営法許可の取得状況を記録 | 登録フォームに許可情報の入力欄 | Must   |
| CL-4-02 | 許可未取得の店舗に対する注意喚起     | ダッシュボードへの警告表示   | Should |

## 2. 資金決済法対応

### 2.1 前払式支払手段に該当しないことの担保


| ID      | 要件                                               | 実装方法                       | 優先度  |
| ------- | ------------------------------------------------ | -------------------------- | ---- |
| CL-5-01 | ポイントの「販売」機能を持たない（利用者から金銭的対価を受け取ってポイントを発行する機能がない） | ポイント購入機能の実装禁止。設計レビューチェック項目 | Must |
| CL-5-02 | 店舗がプレイヤーからポイントを買い取る機能を持たない                       | 買取に該当するトランザクションの検出・アラート    | Must |
| CL-5-03 | ポイントの法的性質が変わりうる機能追加時に法務レビューを必須化                  | 開発プロセスにコンプライアンスレビューゲートを設定  | Must |


### 2.2 ポイントの所在管理


| ID      | 要件                                     | 実装方法                       | 優先度  |
| ------- | -------------------------------------- | -------------------------- | ---- |
| CL-6-01 | ポイントの所在は常に当社にあることを明確化（店舗が「仕入れる」構造にしない） | データモデルとUIで付与枠とポイント残高を明確に区分 | Must |
| CL-6-02 | 付与枠はポイントそのものではなく、「付与リクエスト可能な権利」として設計   | 用語・表示の統一                   | Must |

## 3. 刑法（賭博罪）リスクの排除

### 3.1 設計原則


| ID      | 要件                               | 実装方法                                  | 優先度  |
| ------- | -------------------------------- | ------------------------------------- | ---- |
| CL-7-01 | 参加費がプレイヤーに直接還元される構造にしない          | ポイント付与フローは参加費から独立（広告料→付与枠→付与リクエスト→承認） | Must |
| CL-7-02 | 「還元率」「等価」等の射幸心をそそる表現を広告・UIに使用しない | NGワードチェック（イベント名・説明文・通知文）              | Must |
| CL-7-03 | 当社以外が発行したポイントとの交換機能を持たない         | 外部ポイント連携の実装禁止                         | Must |

## 4. 個人情報保護

### 4.1 個人情報保護法（日本）


| ID      | 要件                          | 実装方法                     | 優先度  |
| ------- | --------------------------- | ------------------------ | ---- |
| CL-8-01 | 利用目的の明示と同意取得                | アカウント登録時のプライバシーポリシー同意フロー | Must |
| CL-8-02 | 個人情報の第三者提供に関する同意            | 必要に応じた個別同意取得             | Must |
| CL-8-03 | 本人からの開示・訂正・利用停止・消去請求への対応    | 管理画面から対応可能な機能を実装         | Must |
| CL-8-04 | 漏えい等発生時の報告義務（個人情報保護委員会への報告） | インシデント対応フローの整備           | Must |
| CL-8-05 | 安全管理措置の実施                   | C4（セキュリティ要件）で対応          | Must |


### 4.2 GDPR（グローバル展開時）


| ID      | 要件                              | 実装方法                 | 優先度    |
| ------- | ------------------------------- | -------------------- | ------ |
| CL-9-01 | データ処理の法的根拠の明確化（同意・契約履行等）        | プライバシーポリシーの GDPR 対応版 | Should |
| CL-9-02 | データポータビリティ権への対応（ユーザーデータのエクスポート） | アカウント設定からデータエクスポート機能 | Should |
| CL-9-03 | 忘れられる権利への対応（アカウント削除時のデータ消去）     | 削除リクエスト機能 + データ匿名化処理 | Should |
| CL-9-04 | Cookie 同意管理                     | Cookie バナーの実装        | Should |
| CL-9-05 | DPO（データ保護責任者）の指定（必要に応じて）        | 組織体制の整備              | Could  |


### 4.3 eKYC データの取り扱い


| ID       | 要件                                     | 実装方法                    | 優先度    |
| -------- | -------------------------------------- | ----------------------- | ------ |
| CL-10-01 | 身分証画像は暗号化して保存、アクセスログを記録                | AES-256暗号化 + アクセスログ     | Must   |
| CL-10-02 | 身分証画像へのアクセスは最小権限原則で制限                  | RBAC で eKYC 担当者のみアクセス可能 | Must   |
| CL-10-03 | 身分証画像は審査完了後14日間保持し、自動削除。テキストデータ（身分証番号・有効期限等）のみ永続保持 | 保存ポリシーの設定 + 14日後自動削除バッチ | Must   |
| CL-10-04 | eKYCで保管される個人情報（氏名等）は英数字（ローマ字）で保管。日本語での保管はグローバル対応に支障が出るため不可 | データモデルの文字コード・フォーマット規定 | Must   |

| -------- | ----------------------- | ------------------------ | ------ |

## 6. 不正利用対策


| ID       | 要件                                     | 実装方法                            | 優先度    |
| -------- | -------------------------------------- | ------------------------------- | ------ |
| CL-12-01 | 異常なポイント取引パターンの検出（大量トランスファー、短時間での大量利用等） | トランザクション監視 + ルールベースのアラート        | Must   |
| CL-12-02 | 複数アカウントの検出・防止                          | デバイスフィンガープリント + eKYC による本人同一性確認 | Should |
| CL-12-03 | 不正アカウントの凍結機能                           | 管理画面からのアカウント停止操作                | Must   |
| CL-12-04 | 不正取引の巻き戻し（ポイント取消）機能                    | 管理者権限でのトランザクション取消               | Must   |
| CL-12-05 | 反社会的勢力の排除チェック（既存規程に基づく）                | 登録時の確認フロー + 定期スクリーニング           | Should |

| ID       | 要件                                    | 優先度    |
| -------- | ------------------------------------- | ------ |

| CL-13-02 | 違反アラートの一覧・対応ステータス管理                   | Must   |
| CL-13-03 | クラブごとのコンプライアンス評価スコア                   | Should |
| CL-13-04 | 規制値（高額サービス上限等）の全体設定・変更                | Must   |

| CL-13-06 | クラブの一時停止・契約解除操作                       | Must   |
| CL-13-07 | コンプライアンスレポートの自動生成（月次）                 | Should |

## 8. 海外展開時の追加考慮事項

PGOS をグローバル展開する際には、以下の法令・規制への対応が追加で必要となる。


| 地域   | 主な法令・規制                               | 対応方針                     |
| ---- | ------------------------------------- | ------------------------ |
| EU   | GDPR                                  | C4-3（プライバシー）で基盤を整備       |
| ブラジル | LGPD (Lei Geral de Proteção de Dados) | GDPR と類似の対応              |
| 各国   | ギャンブル規制法                              | 各国の法令に応じたポイントシステムの適合性確認  |
| 各国   | マネーロンダリング防止                           | 高額トランスファーの報告義務等          |
| 各国   | 年齢制限                                  | 国ごとの年齢要件への対応（18歳 / 21歳等） |

## 9. エスカレーション方針

以下の事案が発生した場合、PGOS システムはアラートを発し、人的対応にエスカレーションする。


| 事案                | アラートレベル | 対応                 |
| ----------------- | ------- | ------------------ |
| 法令違反の疑い（店舗側）      | 高       | クラブの一時停止検討、法務部門へ報告 |
| 不正なポイント取引の検出      | 高       | アカウント凍結、調査開始       |
| 個人情報漏えいの検知        | 最高      | インシデント対応プロセス発動     |
| メディア/報道機関からの問い合わせ | 高       | 広報・法務部門へエスカレーション   |
| 反社会的勢力の関与疑い       | 最高      | 法務部門・警察へ報告         |
