# PGOS 要件定義 — ポイントシステム要件（ウェブコイン後継）

## 概要

本書は、現行のウェブコイン（POKERWEB COIN）を発展させた PGOS のポイントシステム（仮称: PGOS ポイント）の要件を定義する。

ウェブコインは PGOS 最大の差別化要素であり、競合（Poker Fans / Waitinglist）にはない加盟店共通のポイントエコシステムを構築するための核となる。ただし、風営法・資金決済法・刑法のコンプライアンス要件を満たす設計が不可欠である。

## 1. ポイントシステムの基本設計

### 1.1 法的位置づけ（現行踏襲）

| 原則 | 内容 |
|------|------|
| **販売しない** | プレイヤーや店舗に対してポイントを販売しない。対価を受け取らないため、資金決済法上の「前払式支払手段」に該当しない |
| **付与の主体は当社** | ポイントの付与は合同会社グローバルポイントが行う。店舗が直接付与することはできない |
| **景品ではない** | 選手契約（業務委託契約）に基づく報酬として位置づけ、風営法の景品提供に該当しない設計 |
| **等価禁止** | 遊技チップとポイントを等価にする構造は禁止 |

### 1.2 ステークホルダーと役割

| ステークホルダー | 役割 |
|-----------------|------|
| **当社（グローバルポイント）** | ポイントの発行・付与・精算の管理主体 |
| **登録事業者（クラブ）** | 競技結果の報告（付与リクエスト）、ポイント利用の受付、広告料の支払い |
| **プレイヤー（会員）** | ポイントの受取・利用。選手契約の締結が前提 |

## 2. ポイントのライフサイクル

```
┌─────────────┐    広告料支払い     ┌─────────────┐
│  登録事業者   │ ───────────────→ │   当社       │
│  (クラブ)    │                  │ (GP社)       │
└──────┬──────┘                  └──────┬──────┘
       │                               │
       │ 競技結果報告                     │ 付与枠設定
       │ (付与リクエスト)                  │
       ↓                               ↓
┌──────────────────────────────────────────────┐
│              PGOS ポイントシステム               │
│                                              │
│  付与枠 → 付与リクエスト → 承認 → プレイヤーに付与  │
│                                              │
│  プレイヤー利用 → 店舗売上計上 → 精算(広告料範囲内) │
└──────────────────────────────────────────────┘
       ↑                               ↑
       │ ポイント利用                     │ ポイント受取
       │                               │
┌──────┴──────┐                  ┌──────┴──────┐
│  登録事業者   │                  │  プレイヤー   │
│  (受付側)    │                  │  (会員)      │
└─────────────┘                  └─────────────┘
```

## 3. 機能要件

### 3.1 付与枠管理（当社 → 登録事業者）

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-1-01 | 広告料の支払いに応じて、登録事業者に付与枠（ポイント付与可能額）を設定できる | Must |
| WC-1-02 | 付与枠の残高をリアルタイムで確認できる（当社・登録事業者の両方） | Must |
| WC-1-03 | 付与枠が不足した場合、追加の広告料発注フローを提供する | Must |
| WC-1-04 | 付与枠の有効期限設定（契約期間と連動） | Must |

### 3.2 ポイント付与（登録事業者 → 当社 → プレイヤー）

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-2-01 | 登録事業者がトーナメント結果に基づいてポイント付与リクエストを送信できる | Must |
| WC-2-02 | 登録事業者が認定リングゲーム結果に基づいてポイント付与リクエストを送信できる | Must |
| WC-2-03 | 付与リクエストの自動バリデーション: 選手契約完了済みであること | Must |
| WC-2-04 | 付与リクエストの自動バリデーション: eKYC完了済みであること | Must |
| WC-2-05 | 付与リクエストの自動バリデーション: 18歳以上であること | Must |
| WC-2-06 | 付与リクエストの自動バリデーション: 付与枠の範囲内であること | Must |
| WC-2-07 | 付与リクエストのバッチ送信（トーナメント全入賞者分を一括） | Should |
| WC-2-08 | バリデーション通過後、所定期間内にプレイヤーへのポイント付与を完了する | Must |
| WC-2-09 | 付与リクエストのステータス管理（申請中 / 承認済 / 却下 / 取消） | Must |
| WC-2-10 | 却下時の理由通知（登録事業者へ） | Must |

### 3.3 認定制度

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-3-01 | **認定トーナメント制度**: 当社がトーナメントを認定し、認定グレードに応じた付与割合を設定 | Should |
| WC-3-02 | **認定リングゲーム制度**: 当社がリングゲームを認定し、認定グレードに応じた付与割合を設定 | Should |
| WC-3-03 | 認定グレードごとの付与ルール（遊技チップ数 → ポイント数の変換テーブル）を設定・管理できる | Should |
| WC-3-04 | 遊技チップとポイントが等価にならないことの自動検証 | Must |

### 3.4 ポイント利用（プレイヤー → 登録事業者）

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-4-01 | プレイヤーがポイントでトーナメントエントリー費を支払える | Must |
| WC-4-02 | プレイヤーがポイントでリングゲーム遊技料を支払える | Must |
| WC-4-03 | プレイヤーがポイントで飲食代を支払える | Should |
| WC-4-04 | ポイント利用時に5%の手数料を自動で差し引く（手数料は償却、登録事業者に入らない） | Must |
| WC-4-05 | eKYC未完了のプレイヤーはポイント利用不可（自動ブロック） | Must |
| WC-4-06 | 18歳未満のプレイヤーはポイント利用不可（自動ブロック） | Must |
| WC-4-07 | ポイント利用額の上限設定（月あたり等、コンプライアンス観点） | Should |

### 3.5 プレイヤー間トランスファー

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-5-01 | eKYC完了済みプレイヤー間でのポイント移転 | Should |
| WC-5-02 | 双方の eKYC 完了を自動検証 | Must |
| WC-5-03 | トランスファーの日次/月次上限設定 | Must |
| WC-5-04 | トランスファー履歴の完全なトレーサビリティ | Must |
| WC-5-05 | 不正なトランスファーパターンの検知・アラート | Should |

### 3.6 精算（登録事業者 → 当社）

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-6-01 | プレイヤーが登録事業者でポイント利用した分を、当社から登録事業者へ精算できる | Must |
| WC-6-02 | 精算可能額は広告料支払額の範囲内 | Must |
| WC-6-03 | 精算の勘定科目は「両替」（消費税は含まない） | Must |
| WC-6-04 | 精算リクエストの送信・ステータス管理 | Must |
| WC-6-05 | 精算の自動計算・レポート | Should |

### 3.7 選手契約管理

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-7-01 | プレイヤーがアプリ内で選手契約（業務委託契約）を締結できる | Must |
| WC-7-02 | 契約は一度締結すれば有効（再締結不要） | Must |
| WC-7-03 | 契約ステータスの管理（未契約 / 契約済 / 解約） | Must |
| WC-7-04 | 契約内容の閲覧 | Must |
| WC-7-05 | ポイント付与は契約完了者に限定（システム的に強制） | Must |

### 3.8 大会補助

| ID | 要件 | 優先度 |
|----|------|--------|
| WC-8-01 | プレイヤーが大会補助（宿泊・渡航・トーナメント費用）を申請できる | Should |
| WC-8-02 | 補助金額の規定に基づく自動計算（国内/海外、大会種別により異なる） | Should |
| WC-8-03 | 月額上限の自動チェック | Should |
| WC-8-04 | 申請ステータスの管理（申請中 / 承認済 / 却下 / 振込済） | Should |

## 4. 禁止事項チェック（システム的に強制）

現行のウェブコイン運用ルールに基づき、以下をシステムレベルで防止する。

| ID | チェック内容 | 実装方法 | 優先度 |
|----|------------|---------|--------|
| WC-P-01 | 登録事業者がプレイヤーにポイントを直接付与していないか | 付与フローに当社承認を必須化 | Must |
| WC-P-02 | 遊技チップ＝ポイントとなる等価構造になっていないか | 認定制度のルールテーブルで変換率をチェック | Must |
| WC-P-03 | 高額サービス（トーナメント税別10万円超、リングゲーム税別5万円超）の提供 | エントリー費・バイイン設定時に自動チェック | Must |
| WC-P-04 | 後払い10万円超過 | 後払い記録時に自動警告 | Must |
| WC-P-05 | ポイント利用プレイヤーへの手数料上乗せ | ポイント利用時の金額整合性チェック | Must |
| WC-P-06 | 登録事業者によるポイント買取 | 買取に該当するトランザクションの検出 | Must |
| WC-P-07 | 未成年者のポイント利用 | eKYC の年齢確認と連動 | Must |

## 5. 会計処理ガイドライン

| 取引 | 登録事業者の勘定科目 | 消費税 |
|------|-------------------|--------|
| 広告料の支払い（当社へ） | 広告宣伝費 | 10%含む |
| ポイント付与（外注費としてプレイヤーへ） | — (当社が処理) | — |
| プレイヤーのポイント利用による売上 | 売上 | 10%含む |
| 精算金の受取（当社から） | 両替 | 含まない |

## 6. データモデル（概念）

### 6.1 主要エンティティ

| エンティティ | 説明 |
|-------------|------|
| `PointBalance` | プレイヤーごとのポイント残高 |
| `PointTransaction` | ポイントの全取引記録（付与・利用・トランスファー・手数料・精算） |
| `AllocationQuota` | 登録事業者ごとの付与枠（残高・有効期限） |
| `GrantRequest` | 登録事業者からのポイント付与リクエスト |
| `SettlementRequest` | 登録事業者からの精算リクエスト |
| `PlayerContract` | プレイヤーの選手契約ステータス |
| `CertifiedEvent` | 認定トーナメント・認定リングの設定 |

### 6.2 トランザクション種別

| 種別 | 残高への影響 | 発生タイミング |
|------|-------------|---------------|
| `GRANT` | プレイヤー残高 +N | 付与リクエスト承認時 |
| `USE` | プレイヤー残高 -N | ポイント利用時 |
| `FEE` | プレイヤー残高 -(N*0.05) | ポイント利用時（手数料） |
| `TRANSFER_OUT` | 送信者残高 -N | プレイヤー間トランスファー時 |
| `TRANSFER_IN` | 受信者残高 +N | プレイヤー間トランスファー時 |
| `SETTLEMENT` | — | 精算処理（ポイント残高に影響なし） |
| `EXPIRE` | プレイヤー残高 -N | 有効期限到来時（将来検討） |

## 7. 現行ウェブコインからの変更点（検討事項）

| 項目 | 現行 | PGOS での検討 |
|------|------|--------------|
| 名称 | ウェブコイン (POKERWEB COIN) | PGOS ポイント（仮称）— ブランディング検討要 |
| 発行主体 | WEB COIN CENTER Co.Ltd.（タイ） | 当社（グローバルポイント）に一元化を検討 |
| 手数料率 | 5% | 維持または見直し |
| ポイント有効期限 | 設定なし | 有効期限の導入を検討 |
| 利用上限 | 月100万円（補助申請） | 利用上限の設定を検討 |
| トランスファー上限 | 制限なし（eKYC必須） | 日次/月次上限の導入を検討 |
