1. このシチュエーションの概要

店舗の課題

「当日の参加者数が開始まで読めず、テーブル配置が直前まで決まらない。リングゲームのウェイティングも紙で管理していて、連絡も個別に電話やLINE。事前に参加者を把握でき、効率的にウェイティングを管理したい。」

PGOSでは、プレイヤーのセルフレジストレーション（事前エントリー）をリアルタイムで受付・管理。事前エントリー数を把握して準備に活かせる。トーナメントではオルタネイト（配席待ち）の管理にも対応。ウェイティングリストはアプリからの登録・3回呼び出し＋保留リストに対応し、スタッフの連絡業務を削減。受付期間の設定と受付終了後のスタッフ操作の柔軟性を両立。支払い方法はクレジットカード・現金・ウェブコイン・クラブポイント・チケット・デポジットに対応。

2. ユーザーストーリー

T ストーリー1: 「事前エントリーとオルタネイトを管理してスムーズに開始」

店舗スタッフAは、本日19時開始の「デイリーNLH」のPGOS Manager参加者管理画面を確認。15名が事前エントリー（セルフレジストレーション）済み。うち3名はウェブコイン決済完了、2名はチケット利用、10名は現金（当日店頭払い）。事前エントリー数から2テーブルを準備。開始30分前にカウンターで当日参加の4名をチェックイン + エントリー同時処理で登録。合計19名で全席が埋まるため、追加の2名はオルタネイトとしてチェックイン。スタッフが席を案内してプレイイン。

T ストーリー2: 「受付終了後の遅刻者をデポジット利用で対応」

トーナメント開始後、遅刻したプレイヤーBが来店。受付は既に終了しているが、テーブルにはBのためのチップが置かれている。スタッフはPGOS Managerで手動エントリーを実行。「受付終了後のエントリーです。続行しますか？」のアラートが表示されるが、「続行」を選択。Bはデポジット（与信枠¥30,000確保済み）で後払いを選択しており、エントリー費は退店時にまとめて精算される。

R ストーリー3: 「ウェイティングリストの呼び出しと保留管理」

金曜夜、NLH 1-2テーブルが満席。アプリからウェイティング登録したプレイヤーが5名。プレイヤーCが離席すると、ウェイティングリスト先頭のプレイヤーDに自動でプッシュ通知が送信。1分以内にチェックインがないため、次のプレイヤーEに通知。Dは次の空席で優先的に再呼び出し。しかし3回呼び出しても応答がなく、保留リストに移動。Manager画面に「保留中 — 残り12分」と表示される。Dが来店してスタッフに声をかけると、スタッフが来店確認を行い、呼び出しリスト先頭に差し込み。次の空席でDが最優先で呼び出される。

共通 ストーリー4: 「クラブポイントとクーポンの併用処理」

常連プレイヤーFがトーナメントに来店。エントリー費¥3,000に対し、クーポン「エントリー¥500引」をアプリのQRで提示。スタッフがスキャンし割引適用。クーポン割引後の¥2,500に対してクラブポイント1,000ptを利用。残額¥1,500を現金で支払い。スタッフは支払い方法ごとの内訳を記録。クラブポイント利用分はクラブ内で完結し、PGOS精算の対象外。

4-2. トーナメントの受付状態と制御

受付状態トリガーセルフレジストレーション（アプリ）スタッフ操作（Manager）
受付前トーナメント作成時（デフォルト）不可不可
受付中クラブ指定時刻で自動遷移、またはスタッフ手動変更可能可能
受付終了クラブ設定のクローズ時刻で自動遷移、またはスタッフ手動変更不可（ブロック）可能（アラート付き）
進行中トーナメント開始不可可能（アラート付き）
終了全プレイヤーがプレイアウト、またはスタッフ手動終了不可不可

受付開始タイミングの設定方法

  • 絶対指定: 「2026年3月15日 17:00に受付開始」のような日時指定
  • 相対指定: 「トーナメント開始の2時間前に受付開始」のような相対設定
  • 手動: スタッフがPGOS Managerでステータスを「受付中」に変更

セルフレジストレーション（アプリからのエントリー）は「受付中」の間のみ可能。受付終了後は、スタッフ操作のみでエントリーを登録できる（テーブルにチップを置いて後から来る等のケースに対応）。

6. 運用フロー更新

T シナリオ A: 事前エントリーの多いトーナメント

  1. トーナメント作成時にレジストレーション期間を設定（開始2時間前 〜 レベル8終了時）
  2. 受付開始 → プレイヤーアプリに「エントリーする」ボタンが表示される
  3. セルフレジストレーション受付 → 参加者管理画面に「セルフ」としてリアルタイム追加
  4. 事前エントリー数を確認してテーブル数を決定（例: 18名 → 2テーブル）
  5. 当日来店プレイヤーをカウンターで登録（チェックイン + エントリー同時処理）
  6. テーブル・シートのランダム抽選を実行
  7. 全席が埋まっている場合、追加プレイヤーはオルタネイトとしてチェックイン
  8. バストが発生し席が空いたら、スタッフがオルタネイトリストの順番に従い席を指定してプレイイン
  9. トーナメント開始 → 受付終了時刻でセルフレジストレーション自動ブロック
  10. 遅刻者はスタッフ操作でエントリー（アラート確認後に続行）

T シナリオ B: 当日受付のみのトーナメント

  1. セルフレジストレーションを無効に設定（B1-3-05で受付開始を設定しない、または手動のみ）
  2. 来店プレイヤーを受付でスタッフが登録（チェックイン + エントリー同時処理）
  3. ゲスト登録（アカウント未保有者）にも対応
  4. スタッフが手動で受付終了に変更

R シナリオ C: リングゲーム ウェイティング運用

  1. リングゲームスケジュールを事前に設定（B2-1-04）。プレイヤーアプリに自動公開
  2. テーブルオープン → テーブル情報がプレイヤーアプリに公開
  3. プレイヤーがアプリからウェイティング登録 → Manager のウェイティングリストにリアルタイム追加
  4. テーブルに空席発生 → 先頭のプレイヤーに自動でプッシュ通知
  5. 1分以内にチェックインあり → チップイン → 着席
  6. 1分以内にチェックインなし → 次のプレイヤーに通知。元プレイヤーは次の空席で優先再呼び出し
  7. 3回呼び出し無応答 → 保留リストに移動。Manager画面に残り時間表示
  8. プレイヤーが来店しスタッフに声をかける → スタッフが来店確認 → 呼び出しリスト先頭に差し込み
  9. 15分間スタッフ確認なし → 自動キャンセル
