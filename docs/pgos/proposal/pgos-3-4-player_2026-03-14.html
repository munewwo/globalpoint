1. このシチュエーションの概要

プレイヤーの課題

「いま何レベルで次のブラインドがいくらか確認したい。残り何人か知りたい。アドオンしたいけどディーラーを呼ばないといけない。席替えの通知を聞き逃した。」

PGOSでは、トーナメントの進行状況をリアルタイムでアプリに表示。ブラインドスケジュール、残り人数、平均スタック、賞金情報がいつでも確認できる。アドオンやリエントリーもアプリから申請でき、席替え通知はプッシュ通知で確実に受信。

3-3（来店時）との関係

3-3 のチェックイン完了後、プレイヤーはPlay-in 状態でテーブルに着席。本ドキュメントはそこからエリミネーション（またはトーナメント終了）までの体験を扱う。トーナメント結果の確定・ペイアウトは「3-6. プレイ終了・退店時」で定義。

3-2 で定義したトーナメントプレイヤー状態モデルのうち、プレイ中に関連する遷移:

3. ユーザーストーリー

ストーリー1: 「アプリでトーナメント進行をリアルタイム確認」

プレイヤーAはサンデー杯に参加中。ハンドの合間にアプリを開くと、現在レベル6 / ブラインド 300-600 / アンテ 75 / 残り42名 / 平均スタック 28,000 / 次のブレイクまで12分が一目で確認できる。賞金テーブルを見ると「現在42名 → インマネは18名」と表示。あと24人の脱落でインマネだと分かる。

ストーリー2: 「アドオン申請 → ディーラーがチップを配布」

プレイヤーBはスタックが減ってきた。ブレイク中にアプリで「アドオンする」をタップ。支払い方法（ウェブコイン）を確認して申請完了。ディーラーのFloorアプリに通知が届き、ブレイク終了時にアドオン分のチップが配布される。

ストーリー3: 「席替え通知を受信して移動」

プレイヤーCのテーブルが解散（テーブルブレイク）。アプリにプッシュ通知「テーブル5 → テーブル2 / シート3 に移動してください」が届く。アプリ画面にも新しいテーブル/シート番号が表示され、迷わず移動。

ストーリー4: 「チェスクロックで持ち時間を管理」

プレイヤーDはトーナメント参加時にタイムバンク 5分00秒が付与された（早期エントリーのボーナスで +30秒も加算済み）。ハンドごとに30秒のベースタイムが与えられ、30秒を超えた分だけタイムバンクを消費。リバーで大きなベットに直面し、追加で45秒考えたが、消費されたのは15秒だけ（ベースタイム30秒 + タイムバンク15秒）。アプリには残りタイムバンクが「4:45」とリアルタイム表示される。従来のタイムエクステンションチップなら30秒を丸ごと1枚消費するところ、必要な分だけ使えるので公平。

5. ショットクロック（チェスクロック方式）新規

5-1. 設計思想: Triton Tempo モデル

従来のショットクロックの課題

従来型のショットクロックでは、ベースタイム（例: 30秒）を超えるとタイムエクステンションチップ（通常30秒単位）を消費する。しかし、5秒だけ追加で考えたい場合でも30秒チップを丸ごと1枚消費する必要があり、不公平・非効率という問題があった。また、プレイヤーが意図的に時間を使い切るスローイング（時間の浪費）を助長する側面もあった。

PGOS が採用するチェスクロック方式

2026年3月の Triton Poker Series で導入された Triton Tempo に着想を得たシステム。各プレイヤーにトーナメント全体を通した個人タイムバンクを付与し、ベースタイム超過分を秒単位で消費する。必要な時間だけ使えるため公平性が高く、全体のゲーム進行速度も向上する。

タイムバンクの基本ルール

  • ベースタイム: ハンドごとの各アクションに与えられる基本思考時間（デフォルト: プリフロップ 20秒 / ポストフロップ 30秒、クラブ設定で変更可能）
  • タイムバンク: トーナメント開始時に付与される個人持ち時間の合計（デフォルト: 5分、クラブ設定で変更可能）
  • 消費ルール: ベースタイムを超過した秒数だけタイムバンクから減算（1秒単位）
  • 追加付与: レベルアップ時、ブレイク後、深いラウンド進出時にタイムバンクが加算される（例: 5レベルごとに +30秒）
  • 早期エントリーボーナス: レジストレーション期間の早い段階で登録したプレイヤーに追加タイムバンクを付与（例: Day1開始前登録で +30秒）
  • タイムバンク枯渇時: ベースタイム内にアクションしなければ自動フォールド

5-2. タイムバンク表示（A2-3-07）新規

ID要件詳細優先度Ph
A2-3-07タイムバンク表示トーナメント情報画面に自分の残りタイムバンクを常時表示（分:秒形式）。消費時にはリアルタイムでカウントダウンし、残り30秒以下で警告色（赤）に変化。加算イベント（レベルアップ等）時には「+30秒」等のアニメーションを表示。他プレイヤーのタイムバンク残量は、PGOS Clock（店舗ディスプレイ）に一覧表示される（プライバシー設定によりアプリでの閲覧は任意）新規Must1

5-3. アクション中のカウントダウン（A2-3-08）新規

ID要件詳細優先度Ph
A2-3-08アクション中カウントダウン自分のアクション番が回ってきた際、アプリにベースタイムのカウントダウンを表示。ベースタイム終了後はタイムバンクからのカウントダウンに自動切替（表示色を変更: 青→オレンジ）。カウントダウンは PGOS Clock と同期し、アプリ・店舗ディスプレイ・Floor アプリで同一の残り時間を表示。ベースタイム残り5秒でバイブレーション通知（設定可能）新規Must1
アクション時のタイム表示フロー:
アクション開始 → ベースタイム開始（青表示: 30秒…29…28…）
→ ベースタイム残り5秒（バイブ通知） → ベースタイム 0秒
→ タイムバンクに自動切替（オレンジ表示: 残りバンクからカウントダウン）
→ アクション完了でタイマー停止 → 消費秒数をタイムバンクから減算

10. 検討事項・リスク

項目内容対応方針
チップカウントの公開範囲プレイ中のチップカウントをどこまで公開するか。全員に公開すると戦略に影響ファイナルテーブルのみ公開（デフォルト）。クラブ設定で変更可能
アドオン申請の乱用頻繁なアドオン申請によるスタッフ負荷アドオン可能回数の制限（クラブ設定）。ブレイク中のみ受付（デフォルト）
通知の頻度プレイ中の過度な通知がプレイの妨げになる重要通知（席替え・バスト）のみプッシュ。情報系（ブレイク等）はアプリ内のみ（設定可能）
リアルタイム情報の遅延クロック同期の遅延により実際と異なる情報が表示される5秒以内の同期を目標。最終的にはクロック（PGOS Clock）が正
オフライン時の情報表示ネットワーク切断時にリアルタイム情報が更新されない最後に取得した情報を表示し「オフライン」バッジを付与。席替え通知は復帰後に受信
ショットクロック同期精度アプリとディーラー端末/Clock間でカウントダウンにズレが生じる可能性PGOS Clock（サーバー時刻）を正とし、アプリは補正済み時刻を表示。ズレが1秒以上生じた場合は自動再同期
タイムバンク枯渇後の体験タイムバンクが0のプレイヤーはベースタイムのみで全アクションを行う必要があり、ストレスが高い残り30秒以下でハンド開始時に警告。ファイナルテーブル等の重要局面では追加付与で緩和
ショットクロック非対応トーナメントすべてのトーナメントでチェスクロックを使用するとは限らないクラブ設定で「ショットクロック: 有効/無効」を選択可能。無効時はタイムバンク関連UIを非表示
