1. このシチュエーションの概要

店舗の課題

「イベント情報を効率よく公開し、プレイヤーに来てもらいたい。スケジュール作成の手間を省きたい。SNSやクーポンで集客を強化したい。プレイヤーの声に対応したい。」

PGOSでは、PGOS Managerで情報を1回登録すれば、PGOS Portal（公開Web）とPGOS Player（アプリ）の両方に自動反映。さらにテンプレート・カレンダー操作でスケジュール作成を効率化し、広告・ニュース・クーポン・レビュー返信で集客と顧客満足度を向上させます。

2. ユーザーストーリー

ストーリー1: 「今週のスケジュールをテンプレートから素早く作りたい」

店舗オーナーAは PGOS Manager のカレンダー画面を開き、先週のデイリーNLHを複製して来週分を一括作成。特別トーナメントはテンプレート「ディープスタックNLH」から新規作成し、日時だけ変更。ドラッグ＆ドロップで土曜を日曜に移動して確定。プレイヤーアプリに自動反映。各トーナメントの共有URLをコピーしてXに投稿。

ストーリー2: 「ニュースとクーポンで来店を促進したい」

店舗Bのマネージャーはニュース記事で「3月限定キャンペーン」を投稿し、お気に入り登録プレイヤーにプッシュ通知。同時にクーポン「エントリー¥500引」を作成し、クラブ登録者全員にメッセージで配布。プレイヤーはメッセージを開いてクーポンを取得し、来店時にQRコードで利用。マネージャーはクーポンの利用率をレポートで確認。

ストーリー3: 「レビューに返信し、タグとSNSで新規プレイヤーを獲得したい」

新規開店した店舗Cのオーナーは、店舗ページに#初心者歓迎 #NLHのタグを設定。レビューが投稿されると通知が届き、返信コメントで感謝を伝える。SNS連携でLINE公式アカウント・X・Instagramを登録し、プレイヤーがお気に入り登録するとSNSフォローポップアップが表示。ターゲティング広告で渋谷エリアのNLHプレイヤーに新店オープンを告知。

項目内容
要件IDB1-1-01〜06, B1-7-01〜02, B7-3-06, B7-4-02

追加された要素

  • テンプレート選択: 画面上部に「テンプレートから作成」ドロップダウン。選択するとストラクチャー・ペイアウト・タグが一括反映

  • テンプレート保存: 現在の設定をテンプレートとして保存するボタン

7. 運用シナリオ更新

シナリオ A: 新規開店時の初期設定

  1. PGOS Manager にクラブ登録 → M-STORE-EDIT で基本情報入力
  2. ロゴ・カバー画像・写真ギャラリーをアップロード
  3. タグ管理: #初心者歓迎 #NLH 等のクラブタグを設定。自動タグ（地域・NEW）を確認
  4. SNS連携: LINE公式 / X / Instagram のアカウントを登録。フォロー導線を有効化
  5. テンプレート作成: デイリーNLH用のテンプレート（ストラクチャー・ペイアウト含む）を保存
  6. 定期スケジュールを設定 → 2週間分のイベント自動生成
  7. ニュースで「オープンのご挨拶」記事を投稿、プッシュ通知で配信
  8. ターゲティング広告で近隣エリアのプレイヤーに新店オープンを告知

シナリオ B: 週次の通常運用

  1. M-CALENDAR で来週のカレンダーを確認。定期イベントは自動生成済み
  2. 特別トーナメントはテンプレートから新規作成、または既存トーナメントを複製
  3. トーナメントの共有URLをコピーしてXに投稿
  4. レビュー管理で新着レビューに返信
  5. 会計時にサーマルプリンタでレシートを発行（デジタルレシートは自動でアプリに配信）

シナリオ C: 集客キャンペーン

  1. クーポン「エントリー¥500引」を作成し、クラブ登録者全員にメッセージで配布
  2. トーナメント結果連動で「入賞者にドリンク1杯無料クーポン」を自動付与
  3. ニュースでキャンペーン記事を投稿し、プッシュ通知で配信
  4. プロモーション広告で対象トーナメントをリスト上部にハイライト表示
  5. クーポンの利用率・広告の開封率をレポートで確認し、次回施策を改善

8. 検討事項・リスク新規

項目内容対応方針
SNS API制限X / Instagram / LINE のAPI仕様変更やレート制限により、フォロー状況取得や自動投稿に制約が生じる可能性各APIの仕様を調査し、対応可能な範囲で実装。API変更時に影響範囲を最小化する設計
クーポン不正利用QRコードのスクリーンショット共有、クーポンの不正取得1回限り利用制御（スキャン後に即無効化）、デバイスバインド、QRの有効期限（表示から○分間）を検討
レビューの透明性クラブによる不当なレビュー非表示のリスク非表示はPGOS管理者の審査を必須とする。非表示率が高い店舗にはアラート
広告の表示品質プロモーション広告がユーザー体験を損なう可能性広告枠数の上限設定、「PR」ラベルの明示、ユーザーの非表示設定を提供
サーマルプリンタ互換性プリンタメーカー・機種による接続互換性の問題主要メーカー（EPSON / Star Micronics等）を優先対応。Web Bluetooth APIの対応状況に依存
